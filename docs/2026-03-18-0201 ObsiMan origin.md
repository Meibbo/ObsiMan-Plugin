# Problem
Empecé el proyecto porque estaba insatisfecho con las capacidades nativas de Obsidian para gestionar múltiples propiedades en simultáneo en una gran cantidad de archivo markdown.

Obsidian viene con un core plugin de properties view que te permite ver una lista de todas las propiedades detectadas por la app, ordenada de forma descendente por número de ocurrencias y con un filtro de búsqueda por nombre. Solo te permitía reemplazar el nombre de la propiedad. Bastante útil al principio de mi aventura, pero muy limitado acumulas cientos de notas.
# Inspiration
Encontré inspiración en el desarrollo de Notebook Navigator, el como unificó muchas funciones en un solo plugin dejando obsoletos muchos plugins con cientos de miles de descargas.

Este plugin busca unificar, pulir y mejorar las funciones de los siguientes plugins:
- obsidian core properties view
- notebok navigator
- multi properties
- semantic canvas
- CMS bases
# Vision
Quería una combinación de los queries de bases, administrador de archivos, gestor de propiedades, scripts de importación y una queue list (lista de cambios pendientes) para gran multitud de archivos en un amplio rango de situaciones.
