# Open Risks

## Agent Compliance

Docs are weak rails unless backed by scripts, health checks, and review habits.
The system needs mechanical checks for frontmatter shape, line limits, parent
links, forbidden paths, and source preservation.

## Lossy Summaries

The biggest risk is not a long file. The biggest risk is a short file that
hides the decision tree. Health checks should flag summaries without source
links.

## Skill Bloat

Too many overlapping skills can confuse routing. Health should inspect triggers
and propose modular consolidation without deleting functionality.

## Context Pressure

Low context can cause agents to over-summarize. At thresholds, agents should
write handoff shards and stop rather than compressing major decisions into a
tiny final answer.

## Current V2 Status

This v2 spec is richer than v1 but still should be reviewed by the developer
before becoming the canonical plan for future PKM-AI refinement.
