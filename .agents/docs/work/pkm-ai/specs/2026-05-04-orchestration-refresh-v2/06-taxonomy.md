# Taxonomy

## Top-Level Docs

- `current`: live status, handoff, backlog views.
- `work`: active initiatives.
- `architecture`: rules and decisions.
- `knowledge`: descriptive references and manual.
- `archive`: long-term history, raw sources, superseded material.
- `templates`: reusable frontmatter and note shapes.

## Initiatives

- `hardening`: refactor, tests, stability, current branch work.
- `polish`: visual and UX refinement.
- `v1-stable`: release readiness.
- `pkm-ai`: agent-docs system itself.
- `draft`: incubator for ideas not yet promoted.

`draft` is not mirrored as `archive/draft`. Discarded draft ideas go to
`archive/discarded/`. Superseded real work goes under the real initiative.

## Key Distinctions

- item: a unit of work or question.
- attempt: one approach tried for an item.
- decision: why a choice was made.
- policy: current rule.
- knowledge: what sources say.
- manual: teaching material for the developer.
